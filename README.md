## why?
technical explanations are usually dead. they are academic, dry, and lack the friction of real-world operation. **cat explainer** turns systems into lived experiences. it doesn't tell you what a tool *is*; it shows you how kitty dev uses it to win, and how she fails when a config is wrong.

---

## core principles (v1.0)
- **reality first**: use real tools (docker, k8s, redis). no "kitchen" metaphors for servers.
- **minimal metaphor**: use metaphors only for action and pain (e.g., "too many paws").
- **integrated code**: code is an action. it belongs inside the story.
- **tone reset**: lowercase, casual, senior dev field notes. zero fluff.

## installation
add skills via `npx skills` CLI. you can install them all at once or pick what you need.

### install all skills (recommended)
```bash
npx skills add 8w6s/cat-explainer
```

### install individually
```bash
# core engine and coding instincts
npx skills add 8w6s/cat-explainer --skill cat-explainer
npx skills add 8w6s/cat-explainer --skill cat-coder
npx skills add 8w6s/cat-explainer --skill cat-hunt

# specialists
npx skills add 8w6s/cat-explainer --skill cat-network
npx skills add 8w6s/cat-explainer --skill cat-storage
npx skills add 8w6s/cat-explainer --skill cat-debug
npx skills add 8w6s/cat-explainer --skill cat-concept
```
